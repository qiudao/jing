# Tic-Tac-Toe RL

Go 实现的井字棋游戏，使用 Q-Learning 强化学习训练 AI。

## 功能

- 人 vs 人、人 vs AI、AI vs AI 观战
- 三个难度等级（初级/中级/高级）
- 训练面板：进度条 + 收敛曲线
- 慢速观战训练过程
- 模型持久化，重启自动加载

## 运行

```bash
make build
make run
# 打开 http://localhost:8080
```

## 使用流程

1. 在训练面板选择难度，点击 Start Training
2. 训练完成后，选择模式和难度，点击 Start 开始对弈

## 技术栈

- Go + gorilla/websocket
- Q-Learning (epsilon-greedy, temporal difference)
- 纯 HTML/CSS/JS 前端，embed.FS 打包为单一二进制

## Q-Learning 原理

### 核心思想

想象你在教一个完全不会下棋的小孩。他不知道任何规则策略，只能通过**反复下棋、输赢反馈**来自己摸索出好的走法。这就是强化学习。

### 三个关键概念

**状态（State）** — 棋盘当前的样子。比如 `"120000000"` 表示左上角是 X，旁边是 O，其余为空。井字棋总共约 5,478 个合法状态。

**动作（Action）** — 在某个状态下，可以落子的位置（0-8 号格子中空着的那些）。

**Q 值（Quality Value）** — 一张大表，记录「在某个状态下，走某一步有多好」：

```
Q["120000000"][4] = 0.85   ← 在这个局面下，走中间，价值 0.85
Q["120000000"][2] = 0.31   ← 走第2格，价值只有 0.31
```

AI 下棋时就查这张表，**挑 Q 值最高的格子走**。

### Epsilon-Greedy 策略

每一步有两种选择：

- 以概率 ε（探索）→ 随机走一步（尝试新走法）
- 以概率 1-ε（利用）→ 走 Q 值最高的一步（用已有经验）

训练初期 ε=1.0（100% 随机），随训练推进衰减到 0.01（几乎全靠经验）。先广泛尝试，再逐渐收敛到最优策略。

### Q 值更新公式

```
Q(s,a) = Q(s,a) + α × [reward + γ × maxQ(s') - Q(s,a)]
```

| 符号 | 含义 | 本项目取值 |
|------|------|-----------|
| `Q(s,a)` | 在状态 s 走动作 a 的当前评分 | 表里查到的值 |
| `α` | 学习率，新经验的权重 | 0.1 |
| `reward` | 即时奖励 | 赢+1，输-1，平+0.5 |
| `γ` | 折扣因子，未来价值的权重 | 0.9 |
| `maxQ(s')` | 下一个自己回合时，最好的 Q 值 | 往前看两步 |

赢棋的走法会被逐步提高 Q 值，输棋的走法会被降低。经过几万局，好走法浮上来，差走法沉下去。

### 时序差分（TD）学习

关键在于 `γ × maxQ(s')` 这一项。价值像水一样**从终局向前回流**——最后赢棋的那步 Q 值最高，倒数第二步次之，以此类推。这让 AI 能区分关键好棋和无关紧要的步骤。

### 难度等级

三个难度本质上是同样的训练，不同的使用方式：

| 难度 | 训练量 | 对弈时探索率 ε | 效果 |
|------|--------|---------------|------|
| 初级 | 1 万局 | 0.5 | 一半随机走，经常犯错 |
| 中级 | 5 万局 | 0.2 | 偶尔随机，基本理性 |
| 高级 | 20 万局 | 0.0 | 永远走最优，几乎无敌 |

## 强化学习简史

### 起源：动物学习实验（1900s-1950s）

**巴甫洛夫（1900s）** — 狗听到铃声会分泌唾液（条件反射）。环境信号 → 预期奖励。

**桑代克（1911）** — 猫从迷笼逃出拿食物。反复尝试后，成功的动作被强化，失败的被遗忘。他称之为**"效果律"**：带来好结果的行为会被重复。这就是强化学习的核心直觉。

**斯金纳（1930s）** — 系统化研究了奖励/惩罚如何塑造行为。

### 数学化：贝尔曼方程（1957）

**理查德·贝尔曼** 提出了关键的数学框架：

> 一个最优决策可以分解为：当前最优选择 + 从下一个状态开始的最优决策

```
V(s) = max_a [ R(s,a) + γ × V(s') ]
```

这就是 Q-Learning 公式的祖先。他也发明了"动态规划"这个名字——据他自己说，起这个名字纯粹是为了让官僚觉得高深，不砍他的研究经费。

### 时序差分学习（1988）

**萨顿（Richard Sutton）** 提出 TD Learning。关键突破：**不用等最终结果，每走一步就能学习**。之前的方法（蒙特卡洛）必须等一局结束才能更新。

### Q-Learning 诞生（1989）

**克里斯·沃特金斯（Chris Watkins）** 在剑桥大学博士论文中提出 Q-Learning，把贝尔曼方程和 TD 学习结合。**这就是本项目使用的算法。**

为什么叫 Q？Watkins 说 Q 代表 "Quality"——动作的质量。1992 年证明了 Q-Learning 在一定条件下**保证收敛到最优策略**。

### 第一个 RL 棋手：TD-Gammon（1992）

**特萨罗（Tesauro）** 用 TD 学习 + 神经网络做了 TD-Gammon，达到西洋双陆棋世界级水平。但输入仍然是人工设计的特征，不是原始像素。而且这个成功 20 年没能复制到其他领域。

### 从像素学习的梦想

传统做法是人工告诉 AI 该看什么（棋子位置、速度、角度）。长期梦想是：**让 AI 直接看屏幕，自己搞清楚该注意什么。**

实现这个梦想需要三条线索汇合：

**视觉神经科学（1962）** — Hubel & Wiesel 发现猫的视觉皮层分层处理：边缘 → 形状 → 物体。这启发了卷积神经网络（CNN）的设计。

**卷积神经网络（1989）** — LeCun 的 LeNet 用 CNN 从像素识别手写数字。证明了 CNN 能直接从像素学到有用特征，但当时算力有限。

**深度学习爆发（2012）** — AlexNet 在 ImageNet 图像识别比赛中碾压传统方法，错误率从 26% 降到 16%。GPU 的普及是关键推动力。

### DQN：三条线索汇合（2013）

**DeepMind** 做了一件概念简单但之前没人成功的事：

```
CNN（从像素提取特征） + Q-Learning（通过奖励学策略） = DQN
```

为什么之前没人做到？不是没想到，是做不稳定。DeepMind 发现了两个让它稳定的技巧：

- **经验回放** — 把经历存起来随机抽样学习，打破数据相关性
- **目标网络** — 用两个网络，一个更新一个稳定，防止目标漂移

DQN 直接从游戏画面像素学会了 49 款 Atari 游戏，多数超越人类水平。在 Breakout（打砖块）中甚至自己发现了把球打到砖墙后面让它自动弹的策略。

### Q-Table vs DQN

```
Q-Table:    状态 → 查表 → Q值     精确，但状态必须有限
DQN:        像素 → CNN → Q值      近似，但可处理任意输入
```

井字棋只有 ~5,000 个状态，用 Q-Table 精确又快。围棋有 10^170 个状态（比宇宙原子数还多），必须用神经网络。

### AlphaGo 及之后（2016-至今）

**AlphaGo（2016）** 击败围棋世界冠军李世石。用了蒙特卡洛树搜索 + 策略网络 + 价值网络，但核心思想一脉相承：通过自我对弈和价值评估来学习。

**AlphaZero（2017）** 更进一步：零人类知识，纯自我对弈，同一个算法学会了围棋、国际象棋、将棋，全部超越人类。

### 时间线

```
1911  桑代克"效果律" ——————— 好结果的行为会被重复
1957  贝尔曼方程 ———————————— 最优决策的数学分解
1988  TD Learning ————————— 边走边学，不用等结局
1989  Q-Learning ←←←←←←←←← 本项目使用的算法
1992  TD-Gammon ——————————— 第一个世界级 RL 棋手
2012  AlexNet ———————————— CNN 从像素识别图像，碾压传统方法
2013  DQN ——————————————— CNN + Q-Learning，从像素学会 49 款游戏
2016  AlphaGo ——————————— 击败围棋世界冠军
2017  AlphaZero ————————— 零知识自我对弈，通杀棋类
```

本项目用的 Q-Table 是 1989 年的经典算法。它虽然简单，但包含了从贝尔曼到 AlphaZero 一脉相承的核心思想：**让 AI 通过试错和价值评估，自己学会策略。**
